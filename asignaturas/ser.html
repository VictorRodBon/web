CONFIGURAR UNA RED

  1. Comprobar los nombres de las interfaces de red del host:
    ls -l /sys/class/net
  2. Configurar interfaces de red:
    sudo nano /etc/network/interfaces
  3. Contenido:
    auto [nombre de interfaz]
    interface [nombre de interfaz] inet static
    address [dirección ip]
    netmask [máscara de red]
  4. Aplicar cambios:
    /etc/init.d/networking restart
  5. Comprobar estado:
    /etc/init.d/networking status
  6. Activar ip_forwuarder editando el fichero /etc/sysctl.conf:
    Descomentar net.ipv4.ip_forward=1
  7. Activar nat:
    sudo iptables -t nat -A POSTROUTING -o [nombre de interfaz] -j MASQUERADE
  8. Incluir el comando anterior en /etc/rc.local .
  9. Configurar la red en otra máquina (ambas máquinas en red interna).
  10. Comprobar que se encuentran en la misma red:
    ping [ip máquina]

CONFIGURAR UN DHCP

  1. Descargamos el servicio:
    1. Actualizamos los repositorios del sistema
      sudo apt-get update
    2. instalamos el servicio DHCP
      sudo apt-get install isc-dhcp-server
  2. Seleccionamos la interfaz de red que vamos a utilizar:
    1. Editamos el fichero /etc/default/isc-dhcp-server
      Añadimos a "INTERFACES" el nombre de la interfaz
  3. Definimos la red:
    1. Añadir al fichero /etc/dhcp/dhcpd.comf
      subnet [ip de la red] netmask [mascara de red]{
        range [1ª ip del rango] [última ip del rango];
        option domain-name-servers [dirección del DNS];
        option domain-name [nombre del DNS];
        option routers [ip del roputer [en nuestro caso la del servidor]];
        default-lease-time [tiempo de vida predeterminado];
        max-lease-time [tiempo de vida máximo];
      }
  4. Reiniciamos el servicio:
    5. /etc/init.d/idc-dhcp-server restart
  5. Comprobamos el estado del servicio:
    6. /etc/init.d/idc-dhcp-server status
  6. Configuramos un equipo dentro de la misma red para que se le asigne una dirección IP mediante DHCP.
  7. Comprobamos que la IP asignada se encuentra dentro del rango asignado:
    8. ipconfig
  8. Asignamos una direcicón IP fija desde el DHCP:
    1. Comprobamos la MAC de un ordenador:
      ipconfig /all
    2. Editamos el fichero /etc/dhcp/dhcpd.conf:
      host [un nombre]{
       hardware ethernet [dirección MAC];
       fixed-address [dirección ip];
      }
    3. Reiniciamos el servicio:
      /etc/init.d/isc-dhcp-server restart
    4. Comprobamos el estado del servicio:
      /etc/init.d/isc-dhcp-server status
  9. Comprobar que la ip se ha asignado correctamente:
    ipconfig /renew
  10. Observar los paquetes que se envían mediante ireshark:
    1. Discover:
      • Manda un mensaje al broadcast en busca de un servidor DHCP.
      • Ethernet II:
        Manda un mensaje desde la dirección MAC [MAC del host] buscando un servidor DHCP que pueda asignarle una dirección IP.
      • IP:
        Identifica la versión de IP utilizada.
      • Bootstrap Protocol:
        Identifica las características del Discover.
        • Option 53:
          Identifica el tipo de mensage.
        • Option 116:
          Indica que la IP se autoconfigura.
        • Option 61:
          Identificador del cliente.
        • Option 50:
          Indica que IP utilizó por última vez.
        • Option 12:
          Indica el nombre del equipo.
        • Option 60:
          Indentificación de clase del proveedor.
        • Option 55:
          Indica diversos parámetros.
        • Option 43:
          Información específica del proveedor.
        • Option 255:
          Indica el final del paquete.
    2. Offer:
      • Un servidor DHCP recibe el mensaje y manda una IP que puede usar.
      • Ethernet II:
        Manda un mensaje desde la dirección MAC [MAC router] buscando un cliente DHCP.
      • IP:
        Identifica la versión de IP utilizada.
      • Bootstrap Protocol:
        Identifica las características del Discover.
        • Option 53:
          Identifica el tipo de mensage.
        • Option 54:
          Identificador del servidor DHCP.
        • Option 51:
          Tiempo que tarda en volver a mandar una ip.
        • Option 1:
          Indica la máscara de subred.
        • Option 15:
          Indica el nombre del dominio.
        • Option 3:
          Indica la IP del servidor.
        • Option 6:
          Indica el nombre del servidor.
        • Option 255:
          Indica el final del paquete.
    3. Request:
      • El host recibe la IP que le ha mandado el servidor y le manda un mensaje de respuesta.
      • Ethernet II:
        Manda un mensaje desde la dirección MAC [MAC del host] buscando un servidor DHCP.
      • IP:
        Identifica la versión de IP utilizada.
      • Bootstrap Protocol:
        Identifica las características del Discover.
        • Option 53:
          Identifica el tipo de mensage.
        • Option 61:
          Identificador del cliente.
        • Option 50:
          Indica que IP utilizó por última vez.
        • Option 54:
          Identificador del servidor DHCP.
        • Option 12:
          Indica el nombre del equipo.
        • Option 81:
          Nombre completo del dominio.
        • Option 60:
          Indentificación de clase del proveedor.
        • Option 55:
          Indica diversos parámetros.
        • Option 43:
          Información específica del proveedor.
        • Option 255:
          Indica el final del paquete.
    4. ACK:
      • El servidor DHCP recibe un mensaje de respuesta del host y manda un mensaje de finalización de conversación (este mensaje solo se manda si se ha aceptado la dirección IP).
      • Ethernet II:
        Manda un mensaje desde la dirección MAC [MAC del host] buscando un servidor DHCP.
      • IP:
        Identifica la versión de IP utilizada.
      • Bootstrap Protocol:
        Identifica las características del Discover.
        • Option 53:
          Identifica el tipo de mensage.
        • Option 54:
          Identificador del servidor DHCP.
        • Option 51:
          Tiempo que tarda en volver a mandar una ip.
        • Option 1:
          Indica la máscara de subred.
        • Option 15:
          Indica el nombre del dominio.
        • Option 3:
          Indica la IP del servidor.
        • Option 6:
          Indica el nombre del servidor.
        • Option 255:
          Indica el final del paquete.

CONFIGURAR UN DNS

  1. Instalamos el servicio Bind9:
    sudo apt-get install bind9
  2. Lo configuramos como servidor caché:
    Se configra automáticamente.
  3. Comprobamnos que funciona correctamente:
    nslookup localhost
  4. Configuramos el DHCP para que utilice el servici que acabamos de instalar como DNS por defecto:
    subnet [ip de la red] netmask [mascara de red]{
      range [1ª ip del rango] [última ip del rango];
      option domain-name-servers [dirección del servidor];
      option domain-name [nombre del DNS];
      option routers [ip del roputer [en nuestro caso la del servidor]];
      default-lease-time [tiempo de vida predeterminado];
      max-lease-time [tiempo de vida máximo];
    }
  5. Reiniciamos el servicio:
    /etc/init.d/bind9 restart
  6. Comprobamos el estado del servicio:
    /etc/init.d/bind9 status
  7. Editamos el fichero /etc/bind/named.conf.options y descomentamos forwarders y le indicamos el que queremos.
    forwarders{
      [dirección del servidor que quieremos que sea usado];
    }
  8. Reiniciamos el servicio:
    /etc/init.d/bind9 restart
  9. Comprobamos el estado del servicio:
    /etc/init.d/bind9 status
  10. Comprobamos que el forwarder funciona:
    dig [primer dominio] [IP ervidor root]
    dig [segundo dominio] [primera IP resultante del dig anterior] (Repetir los pasos hasta que no salgan direcciones y salgan dominios)
    dig [primer dominio completo] [IP ervidor root]
    dig [dominio original] [IP resultante del dig anterior] (obtendremos una única dirección IP)
  11. Creamos la zona directa editando el fichero /etc/bind9/named.conf.local:
    zone "[nombre de la zona]"{
     type master;
     file "[nombre del fichero en el que se almacena la información de la zona directa]";
    }
  12. Copaiamos el fichero /etc/bind9/db.empty y le damos el nombre que hemos indicado en el apartado anterior.
  13. Editamos el fichero que acabamos de crear:
    @ IN SOA ns1.[nombre de dominio]. [usuario].[nombre de dominio].(
    )
    [dominio] IN NS ns1.[nombre de dominio].
    [dominio] IN A [IP del dominio]
    www IN CNAME [nombre del dominio]
  • Significado de los parámetros
    • SOA:
    Registro de autoridad.
    Almacena la información de un dominio o zona.
    IN:
    Indica la arquitectura de protocolos utilizada.
    NS:
    Determina los servidores que comunicacn la información del DNS.
    A:
    Vincula un dominio con una dirección IPv4.
    AAAA:
    Vincula un dominio con una dirección IPv6.
    CNAME:
    Enlaza el nombre de un dominio con un alias.
  1. Reiniciamos el servicio:
    /etc/init.d/bind9 restart
  2. Comprobamos el estado del servicio:
    /etc/init.d/bind9 status
  3. Realizamos 2 consultas al ervidor, una dentro de nustra red y otra fuera:
    nslookup [nombre de dominio] [dirección IP del servidor]
  4. Creamos la zona directa editando el fichero /etc/bind9/named.conf.local:
    zone "[nombre de la zona]"{
     type master;
     file "[nombre del fichero en el que se almacena la información de la zona directa]";
    }
  5. Copaiamos el fichero /etc/bind9/db.empty y le damos el nombre que hemos indicado en el apartado anterior.
  6. Editamos el fichero que acabamos de crear:
    @ IN SOA ns1.[nombre de dominio]. [usuario].[nombre de dominio].(
    )
    [IP puesta al reves sib el últinmo octerto]-addr.arpa. IN NS ns1.[nombre de dominio].
    [dominio] IN PTR [Nombre del dominio]
  • Significado de los parámetros
    • SOA:
    Registro de autoridad.
    Almacena la información de un dominio o zona.
    IN:
    Indica la arquitectura de protocolos utilizada.
    NS:
    Determina los servidores que comunicacn la información del DNS.
    PTR:
    Vincula una dirección IPv4 con un nombre de dominio.
  1. Reiniciamos el servicio:
    /etc/init.d/bind9 restart
  2. Comprobamos el estado del servicio:
    /etc/init.d/bind9 status
  3. Realizamos consultas al servidor:
    nslookup [IP dominio] [IP servidor]

CONFIGURAR UN FTP

  1. Actualizamos los repositorios del sistema:
    sudo apt-get update
  2. Instalamos el servicio:
    Sudo apt-get install vsfptd.
  3. Iniciamos el servicio:
    /etc/init.d/vsftpd start
  4. Entramos con un Windows XP:
    ftp [ip del servidor]
  5. Habilitamos el modo anónimo:
    Entramos en el documento /etc/vsfpt.conf y descomentamos "anonymous_enable=yes".
  6. Reiniciamos el servicio:
    /etc/init.d/vssftpd restart
  7. Comprobamos el estado del servicio:
    /etc/init.d/vssftpd status
  8. Comprobamos desde un equipo con Windows xp que se ha habilitado
    ftp [ip del servidor]
    anonimous
  9. Habilitamos subir archivos en modo autentificado:
    Editamos el fichero /etc/vsftpd.conf y descomentamos:
      local_enable=yes
      write_enable=yes
     local_umask=022
  10. Reiniciamos el servicio:
    /etc/init.d/vsftpd restart
  11. Comprobamos el estado del servicio:
    /etc/init.d/vsftpd status
  12. Comprobamos desde otro equipo e intentamos subir un fichero al servidor:
    ftp [ip del servidor]
    [Usuario]
    put [fichero]
  13. Creamos las carpetas y los ficheros que los usuarios verán cuando entren en modo anónimo:
    mkdir /srv/ftp/[directorio]
    mkdir /srv/ftp/[directorio]
  14. Comprobamos que se han añadido desde otro equipo
    ftp [ip del servidor]
    anonimous
    ls
  15. Editamos el fichero /etc/vsftpd.conf y descomentamos:
    chrot_local_users=yes
    allow_writeable_chroot=yes
    chroot_list_enable_yes
    chroot_list_file=/etc/vsftpd.chroot_list
  16. Reiniciamos el servicio:
    /etc/init.d/vsftpd restart
  17. Comprobamos el estado del servicio:
    /etc/init.d/vsftpd status
  18. Introducimos en el fichero /etc/vsftpd.chroot_list los usuarios que queremos que tengan libertad de movimiento
    Creamos un usuario
      sudo adduser [usuario]
    Añadimos el usuario al fichero.
  19. Reiniciamos el servicio:
    /etc/init.d/vsftpd restart
  20. Comprobamos el estado del servicio:
    /etc/init.d/vsftpd status
  21. Comprobamos que se ha aplicado desde otro equipo
    ftp [ip del servidor]
    [usuario enjaulado (no se ha introducido en el fichero)]
    pwd (tiene que salir que se encuentra en un directorio raid)
  22. Entramos en modo activo:
    ftp [ip del servidor]
    [usuario]
    Comprobamos que los puertos utilizados son 21 y 20
  23. Entramos en modo pasivo:
    Entramos desde Fillecilla
    [usuario]
    Comprobamos que los puertos utilizados son 20 y uno superior al 1023
